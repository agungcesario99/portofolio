Projects

Mobile · SDK Integration

Implementing Liveness SDK in Mobile

Replaced the legacy Base64-over-API liveness flow on IPOS Mobile by integrating a Passive Liveness SDK plugin. The SDK now captures the selfie (front or back camera), performs on-device liveness validation, and returns the result directly to IPOS — which shows success and proceeds, or maps the SDK backend error code to a localised failure message (English / Bahasa).

Mobile · API Integration

Implementing OCR in Mobile

Migrated IPOS from internal OCR to an external OCR API for KTP & Passport data extraction. The captured ID image is converted to Base64 and sent in the request body; the API response auto-populates Name, DOB, Place of Birth, and NIK in the customer form, with users able to edit fields before submission.

Web · SDK Integration

Implementing Liveness SDK in Web

Brought the same Passive Liveness SDK to IPOS Web, replacing the previous Base64 API call. The SDK handles capture and validation in-browser and returns the result to IPOS — success allows the user to proceed, while failure cases are surfaced with localised error messages (English / Bahasa) mapped from the SDK backend error codes.
